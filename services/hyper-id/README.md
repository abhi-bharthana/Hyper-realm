🛸 Hyper-ID: Developer & Systems Manual v1.0Ye manual Hyper-realm Ecosystem ke core identity node (Hyper-ID) ko samajhne aur setup karne ke liye banaya gaya hai.1. 🏗 System Architecture (The Big Picture)Hyper-ID ek Identity Provider (IdP) hai. Ye Google OAuth 2.0 ka use karke user ko verify karta hai aur hamare ecosystem ke liye ek custom JWT (JSON Web Token) issue karta hai.Auth Flow: Google Login -> Backend Verification -> User Creation (if new) -> Onboarding (if status is pending) -> Final JWT.SSO Capability: Ek baar login hone par, user client_id ke zariye kisi bhi "Hyper" project (Transit, Fuel) mein redirect ho sakta hai.2. 📁 Folder Structure & Package LogicNaye dev ke liye map ye raha:cmd/server/: Main entry point. Server yahan se bootstrap hota hai.internal/api/: Handlers. HTTP requests (Login, Onboarding) yahan handle hoti hain.internal/auth/: Security. JWT creation, verification, aur Google token validation ka logic.internal/db/: Database Layer. Postgres connection aur raw SQL queries.internal/models/: Structs. Go objects jo database tables ko represent karte hain.certs/: Secret Keys. RS256 signature ke liye RSA private/public keys.3. 🔐 Environment Variables (.env)Project root mein ek .env file honi chahiye. Iska har variable zaroori hai:VariableDescriptionPORTServer kis port par chalega (Default: 8080).DB_URLPostgres connection string (postgres://user:pass@localhost:5432/dbname).GOOGLE_CLIENT_IDGoogle Cloud Console se mila hua unique ID.JWT_PRIVATE_KEY_PATH./certs/private.pem ka path.JWT_PUBLIC_KEY_PATH./certs/public.pem ka path.4. 🔑 Certificates Setup (The RSA Brain)Hyper-ID RS256 use karta hai. Iska matlab hum token sign karne ke liye private key aur verify karne ke liye public key use karte hain.Naya Key-pair generate karne ke liye commands:ShutterstockBash# Private key generate karo
openssl genrsa -out certs/private.pem 2048

# Public key extract karo
openssl rsa -in certs/private.pem -pubout -out certs/public.pem
5. 🗄 Database & MigrationsHum Docker mein Postgres use kar rahe hain. Naye system par table setup karne ke liye ye steps follow karein:Step 1: Start DatabaseBashdocker-compose up -d
Step 2: Run Migrations (Manual)Terminal mein ye commands chala kar tables create karein:Bash# 1. Users Table
docker exec -it hyper_id_db psql -U abhi -d Hyper-ID -c "CREATE TABLE h_users (hid UUID PRIMARY KEY DEFAULT uuid_generate_v4(), email TEXT UNIQUE, username TEXT UNIQUE, account_status TEXT DEFAULT 'pending_onboarding');"

# 2. Profiles Table
docker exec -it hyper_id_db psql -U abhi -d Hyper-ID -c "CREATE TABLE h_profiles (hid UUID REFERENCES h_users(hid), first_name TEXT, last_name TEXT, nickname TEXT);"

# 3. Clients Table (For SSO)
docker exec -it hyper_id_db psql -U abhi -d Hyper-ID -c "CREATE TABLE h_clients (client_id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), client_name TEXT, client_secret TEXT, redirect_uri TEXT);"
